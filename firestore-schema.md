# Voqua Firestore Database Schema

## Overview

This document describes the Firestore database structure for the Voqua teacher-student classroom system.

---

## Collections

### `users/{uid}`

Stores profile information for all users. The `uid` matches the Firebase Auth UID.

| Field | Type | Description |
|-------|------|-------------|
| `role` | `string` | `"teacher"` or `"student"` |
| `fullName` | `string` | User's full name |
| `dob` | `string` | Date of birth (e.g. `"YYYY-MM-DD"`) |
| `school` | `string` | School name |
| `grade` | `string` | Grade level — **students only** |
| `subject` | `string` | Subject taught — **teachers only** |
| `createdAt` | `timestamp` | Firestore server timestamp at account creation |

**Example document path:** `users/abc123uid`

---

### `classes/{classId}`

Stores classroom information created by teachers. The `classId` is auto-generated by Firestore.

| Field | Type | Description |
|-------|------|-------------|
| `teacherUid` | `string` | UID of the teacher who owns this class |
| `className` | `string` | Display name of the class |
| `subject` | `string` | Subject of the class |
| `school` | `string` | School where the class is held |
| `classCode` | `string` | 6-character random join code (e.g. `"A3F9KZ"`) |
| `studentUids` | `array<string>` | Array of UIDs for enrolled students |
| `createdAt` | `timestamp` | Firestore server timestamp at class creation |

**Example document path:** `classes/xyz789classId`

---

### `assignments/{assignmentId}`

Stores assignments created by teachers for a specific class. The `assignmentId` is auto-generated by Firestore.

| Field | Type | Description |
|-------|------|-------------|
| `classId` | `string` | ID of the class this assignment belongs to |
| `teacherUid` | `string` | UID of the teacher who created the assignment |
| `title` | `string` | Assignment title |
| `instructions` | `string` | Instructions shown to students |
| `script` | `string` | The reading/speaking script students must perform |
| `dueDate` | `timestamp` | Due date for the assignment |
| `createdAt` | `timestamp` | Firestore server timestamp at assignment creation |

**Example document path:** `assignments/asgn001Id`

---

### `submissions/{submissionId}`

Stores a student's submission for an assignment. The `submissionId` is auto-generated by Firestore.

| Field | Type | Description |
|-------|------|-------------|
| `assignmentId` | `string` | ID of the assignment being submitted |
| `studentUid` | `string` | UID of the student who submitted |
| `transcript` | `string` | AI-generated transcript of the student's recording |
| `aiScore` | `number` | AI-generated score (e.g. `0`–`100`) |
| `recordingUrl` | `string` | Temporary URL to the audio recording — deleted after grading |
| `teacherGrade` | `string` | Grade assigned by the teacher (e.g. `"A"`, `"85"`) |
| `teacherComment` | `string` | Teacher's written feedback |
| `status` | `string` | `"submitted"` or `"graded"` |
| `submittedAt` | `timestamp` | Firestore server timestamp when student submitted |
| `gradedAt` | `timestamp` | Firestore server timestamp when teacher graded |

**Example document path:** `submissions/sub001Id`

---

## Security Notes

- `recordingUrl` is temporary and should be deleted from storage once grading is complete.
- Firestore Security Rules should enforce that:
  - Students can only read/write their own `submissions` documents.
  - Teachers can only read submissions for assignments in their own classes.
  - Only teachers can create/update `assignments` and `classes`.
  - Users can only read/write their own `users/{uid}` document.
